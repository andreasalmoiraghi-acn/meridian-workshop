R2 — Key New Capability

Restocking recommendations

A new view that surfaces purchase recommendations based on current stock levels, demand forecast, and an operator-supplied budget ceiling.

Before writing a line of code, we run a structured session with your Operations team to document the business rules: reorder thresholds, demand weighting, priority logic. The feature is built to those rules — not to our assumptions.

Operator sets budget

The restocking view accepts a budget ceiling as input — recommendations are ranked and filtered accordingly.

Integrates existing data

No new data sources required. The feature uses inventory, demand forecast, and backlog data already in the system.

Validated early

A working prototype (M4, week 7) is reviewed with Operations before final delivery — reducing rework risk.

D1 · D2 · D3 — Desired Items

Extensions, not additions

The desired items are sequenced as natural extensions of required work — delivered in the same phases, not as a separate track.

D1

UI Modernization

A visual proposal (palette, typography, layout) is presented in week 1–2 and approved before any implementation begins. Applied via the existing design token system — minimal regression risk.

D2

Internationalization

i18n infrastructure is already live for 5 views. We extend it to Reports and Restocking, and add Japanese for the Tokyo team. Translation glossaries provided by Meridian.

D3

Dark Mode

Developed on an isolated branch — zero risk to production until validation. Operator-selectable theme, persisted per workstation. Ideal for low-light warehouse environments.
